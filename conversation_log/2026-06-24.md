## 错误分析

**错误 1：`D:miniconda3python.exe: command not found`**
- UserPromptSubmit hook 里 python 路径用了 Windows 反斜杠 `D:\miniconda3\python.exe`，但 bash 把 `\m`、`\p` 当转义吃掉了，变成 `D:miniconda3python.exe`
- 修复：路径用正斜杠 `D:/miniconda3/python.exe` 或引号包裹 `"D:\miniconda3\python.exe"`

**错误 2：`API Error: 400 ... server_tool_use ...`**
- 这是 glm-5.2 上游 API 的问题，它不认 `server_tool_use` 这个 content type（Claude 官方 API 支持的扩展类型，第三方代理转发时没过滤掉）
- 这个不是我们能在客户端修的，属于中转服务把 Anthropic 原生 tool 协议透传给了 glm
- 上次会话死循环可能就是：工具调用结果带 `server_tool_use` 标签 → API 400 → 重试 → 又带标签 → 又 400

**结论**：错误 1 在 settings.json 里改路径就能修；错误 2 是上游协议不兼容，只能换模型或换中转。两个都不是 bucket-list-ai 项目代码的问题。
